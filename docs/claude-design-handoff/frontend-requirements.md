# Multi AI Mate Frontend Requirements

## 1. Project Goal

Multi AI Mate 是一个桌面端多 AI 工作台。用户可以在一个窗口里同时打开 1 到 4 个 AI 网站，并把同一条消息默认发送给当前模式下所有已打开的 AI 窗口。

当前阶段只做前端界面和交互原型设计。不要实现真实 WebView、登录、注入、自动发送、桌面打包。

## 2. Target Platform

第一阶段优先 Mac 桌面端体验。

设计时要保留未来兼容 Windows 的空间，但当前不需要按 Windows 原生界面重新设计。

## 3. Core Product Rules

### 3.1 AI Mode

界面顶部需要有独立按钮:

- `1 AI`
- `2 AI`
- `3 AI`
- `4 AI`

这些按钮用于切换当前工作台显示几个 AI 面板。

默认发送逻辑:

- `1 AI`: 发送给当前主面板 A
- `2 AI`: 同时发送给 A 和 B
- `3 AI`: 同时发送给 A、B、C
- `4 AI`: 同时发送给 A、B、C、D

不需要单独的“范围”按钮。发送范围始终是当前模式下所有已打开的 AI 面板。

### 3.2 Layout

`1 AI`:

- 主栏 A 最大化
- 适合专注使用一个 AI

`2 AI`:

- A、B 两个主栏左右并排
- 两栏宽度相等

`3 AI`:

- A、B 是两个主栏
- C 是右侧副栏
- C 的宽度约为单个主栏的 80%
- 需要让 C 在视觉上明显是辅助面板，而不是第三个同级主面板

`4 AI`:

- A、B 是两个主栏
- 右侧增加副栏区域
- 副栏区域上下分成 C、D 两个面板
- C、D 共享右侧较窄区域

### 3.3 AI Provider Dock

左侧栏是一列常用 AI 网站 logo 或文字缩写，作为 AI 库。

建议包含:

- ChatGPT
- Claude
- Gemini
- 豆包
- DeepSeek
- Kimi
- Grok
- Custom

左侧 AI 入口的规则:

- 点击 logo 不应该立即替换任何面板
- 只有拖拽 logo 到某个面板时，才把该 AI 加载到对应面板
- 拖拽过程需要有清晰的目标高亮
- 空面板需要提示“拖入 AI 开始”

### 3.4 Panel Rules

每个 AI 面板需要显示:

- 面板标识: A / B / C / D
- 当前 AI 名称
- 当前网页状态，例如 `Ready`、`Empty`、`Loading`
- 网页内容区域的视觉占位

面板上不需要显示以下文字按钮:

- 刷新
- 放大
- 关闭

刷新通过桌面快捷键处理:

- Mac: `Cmd+R` 同时刷新当前模式下所有已打开网页

后续真实产品里可以用图标按钮或菜单隐藏这些能力，但当前原型不要把它们作为显眼文字按钮放在面板头部。

### 3.5 Close and Empty Slot Behavior

如果某个分栏被关闭，关闭的分栏保持为空槽位。

重要:

- 不要自动把其他 AI 面板挪过来
- 不要重新分配面板顺序
- 不要自动恢复被关闭的副栏
- 空槽位保留原来的 A/B/C/D 身份

例如在 `4 AI` 中关闭 C:

- A、B、D 保持原位
- C 显示为空槽位
- 右侧仍然上下双栏，只是 C 为空

### 3.6 Bottom Composer

底部是统一输入区。

需要表达:

- 用户输入一条消息
- 点击发送后，默认发给当前模式下所有已打开 AI 面板
- 输入区不需要“发送范围”切换
- 可以显示当前目标提示，例如: `Will send to all open panels in 3 AI mode`

底部输入区要干净，适合长期写作、提问、对比多 AI 回答。

## 4. Visual Direction

设计风格: 专业、克制、像生产力工具，不要像营销落地页。

关键词:

- 桌面工作台
- 多窗口并排
- 低干扰
- 清晰层级
- Mac 应用感
- macOS 毛玻璃材质
- 半透明侧边栏
- 轻微背景模糊
- 原生桌面软件质感
- 可长时间使用

避免:

- 大面积渐变背景
- 夸张卡片
- 营销式 hero
- 过多说明性文案
- 过度圆角
- 全紫色或单一色系

建议:

- 左侧 dock 使用紧凑图标
- 顶部模式按钮使用 segmented control
- 面板边界清楚但不要沉重
- A/B 主面板更强，C/D 副面板更轻
- 空槽位状态要明显但安静
- 底部输入区要稳定，不随文本或按钮变化跳动

### 4.1 Mac Material Direction

这个界面不应该像普通 HTML 网页，而应该接近 Mac 桌面软件的视觉质感。

设计时请使用:

- `backdrop-filter: blur(...)`
- `-webkit-backdrop-filter: blur(...)`
- 半透明面板背景
- 细边框高光
- 柔和阴影
- sidebar / titlebar / composer 的 glass material
- panel 内部更清晰，避免正文内容被过度透明影响可读性

需要注意:

- 毛玻璃应该提升层次感，不要让界面变花
- 主要阅读区域仍然要清晰稳定
- 透明和模糊效果要有节制
- 不要做成过度梦幻、过度渐变或网页 landing page
- 目标是“真实 Mac 工具 App”，不是“炫技玻璃卡片”

工程实现时，CSS 毛玻璃可以先在前端实现。真正的窗口级 macOS vibrancy、透明窗口、原生标题栏融合，后续由 Codex 在 Electron/Tauri 层处理。

## 5. Prototype Requirements For Claude Design

请基于现有 HTML 原型进行二次设计，不要改变核心产品逻辑。

可以修改:

- 颜色
- 字体
- 间距
- 面板比例
- Dock 样式
- 顶部按钮样式
- 输入区样式
- 空槽位样式
- 拖拽目标状态
- 1/2/3/4 AI 模式视觉切换

不要修改:

- 1/2/3/4 AI 的基本布局关系
- 默认发送到当前模式下所有已打开窗口
- 左侧 logo 只有拖拽才改变面板
- 去掉“范围”按钮
- 面板不显示“刷新/放大/关闭”文字按钮

## 6. Deliverable Expected From Claude Design

请输出:

1. 一个改进后的单文件 HTML 原型
2. 简短说明设计改动
3. 关键组件清单
4. 交给工程实现时需要注意的交互状态

最终 HTML 会交回 Codex，由 Codex 实现成真实前端界面。
